# 치크 숏폼 프로젝트 - AI Agent 지침

## 1. 비즈니스 맥락

### 배경
- **채널**: 크래프톤/배틀그라운드 공식 유튜브
- **발견**: 유튜브 영상 중 **신규 시청자 수가 가장 많은 영상**이 "커스텀 훈련장" 관련 숏츠
- **니즈**: 배그 자체 훈련장 시스템이 부실 → 실력 향상을 원하는 유저들의 니즈 높음

### 목표
- **PWS(PUBG World Series)** 대회 참가 유도
- 댓글 참여 유도 ("3일" 댓글, PWS 신청)
- 커스텀 훈련장 콘텐츠로 신규 시청자 유입

### 타깃
- 배그를 꾸준히 하는데 실력을 올리고 싶은 사람
- "나 좀 한다"는 사람들 (아마추어 → 대회 참가 유도)

### 가설
| 구분     | 내용                        |
| -------- | --------------------------- |
| 성공가설 | 3편 업로드 시 댓글수 증가   |
| 실패가설 | 댓글수 낮으면 공감구조 약함 |
| 후속대응 | '공감 메시지' 변경          |

## 2. 스토리 아크 (3편 연결 구조)

### 전체 흐름
```
[EP01] 게이머가 맨날 죽어서 답답함
       → ChatGPT에게 커스텀 훈련장 의뢰
       → 갑자기 게임 속으로 빨려들어감
       → 치킨 먹어야 탈출 가능
       → 훈련하며 성장
       → 저격당함 (클리프행어: 맞저격 vs 엎드림)

[EP02] 맞저격 성공!
       → 회상: 뉴비 시절 vs 지금
       → 명중률 87% 달성
       → 1명만 더 잡으면 탈출
       → 보급품 발견 (클리프행어: 가져갈까 vs 확인할까)

[EP03] 보급품이 아니라 병아리(치크)였다!
       → 게이머 좌절
       → 치크가 알고보니 숨은 고수
       → 치크의 훈련 루틴 공개
       → PWS 신청 CTA
```

### 핵심 메시지
- "실력은 오래 하는 게 아니라 **루틴**이 필요하다"
- "아마추어도 PWS 참가 가능하다"

## 3. 세계관

### 등장인물

**GAMER (주인공)**
- 배그를 좋아하지만 맨날 죽는 평범한 유저
- 실력 향상을 간절히 원함
- 성격: 열정적, 좌충우돌, 성장형 캐릭터
- 외형: 아시아 남성 20대, 검은 긴팔, 청바지, 헤드셋
- 참조: `assets/게이머.png`

**GPT (로봇 조력자)**
- ChatGPT가 게임 속에서 구현된 형태
- 침착하고 논리적으로 설명
- 성격: 차분함, 로봇다움, 약간의 유머
- 외형: 배그 캐릭터, 레벨3 헬멧(얼굴 가림), 흰 셔츠, 무장
- 대사 스타일: "~합니다" 체
- 참조: `assets/헬멧남.png`

**CHEEK (병아리 조력자)**
- 보급품인 줄 알았더니 병아리
- 알고보니 숨은 고수 (반전)
- 성격: 귀여움, 하지만 실력자
- 외형: 노란 병아리, 군용 사막 위장 헬멧, 주황색 부리와 발
- 대사: "삐약?" (후편집으로 자막 처리)
- 참조: `assets/치크.png`

| ID             | 설명                                         | 사용 에피소드          |
| -------------- | -------------------------------------------- | ---------------------- |
| GAMER_ROOM     | 어두운 게이밍룸, 모니터 불빛만               | EP01 씬1               |
| PUBG_CITY      | 배그 도시, 폐허 건물, 잔해, 회색 하늘        | EP01 씬2-5, EP02 씬1-4 |
| PUBG_FIELD     | 배그 초원, 보급품 낙하, 푸른 존              | EP02 씬5-6, EP03 씬1-2 |
| PUBG_WAREHOUSE | 배그 창고 내부, 철제 선반, 박스, 어두운 조명 | EP03 씬3-6             |

## 5. 기술 스택

### 상위 프로젝트
- **drama-shorts-web**: Django Admin + LangGraph 기반 영상 자동 생성 시스템
- 이 프로젝트는 코드화 전 **프롬프트 실험 단계**

### 영상 생성
| 항목      | 값                    |
| --------- | --------------------- |
| 모델      | Veo 3.1-fast (fal.ai) |
| 해상도    | 1080x1920 (9:16 세로) |
| 씬당 길이 | 2~5초                 |
| 총 길이   | 18~19초               |

### 제약사항
| 항목        | 규칙                         | 이유          |
| ----------- | ---------------------------- | ------------- |
| 한글 텍스트 | 영상 내 생성 금지            | Veo 한글 깨짐 |
| 자막/말풍선 | 후편집 처리                  | 품질 보장     |
| 대사        | JSON의 `audio_reference`에만 | 참고용        |
| 효과음      | 후편집 처리                  | 품질 보장     |
| "삐약" 소리 | 제외                         | 요청사항      |

## 7. 작업 흐름

### 프롬프트 실험 → 코드화 순서

```
1. JSON 프롬프트 작성
   └── prompts/TEMPLATE_GUIDE.md 참고
   └── characters, locations, scenes 구조 준수

2. JSON → Veo 프롬프트 변환
   └── 변환 규칙 적용 (아래 참조)
   └── "no text on screen" 항상 추가

3. 웹 UI에서 수동 테스트
   └── fal.ai 또는 drama-shorts-web Admin

4. 결과 확인 및 기록
   └── TODO.md에 성공/실패 기록
   └── 개선점 도출

5. 반복 (만족할 때까지)

6. 확정 후 코드 구현 (필요시)
   └── drama-shorts-web의 game_services.py 확장
   └── 또는 새 타입 추가
```

### JSON → Veo 프롬프트 변환 규칙

```
{characters[ID].visual} + {setting.location_desc} + {camera 정보} + {action.description} + {mood} + "No text on screen."
```

**예시 (EP01 씬1)**:
```
Young Asian male wearing black long-sleeve shirt, dark blue jeans, and gaming headset sits in a dark gaming room illuminated only by bright monitor light. Medium shot from behind and slightly to the right. A swirling vortex of light bursts from the screen, pulling him toward the monitor. His body stretches as he gets sucked into the digital world. Surreal, dramatic, sci-fi atmosphere. No text on screen.
```

## 8. 캐릭터 영상 생성 규칙

### 캐릭터 구분 (Veo 혼동 방지)

1. **외형 명시**: 매 프롬프트에 캐릭터 visual 전체 포함
2. **위치 명시**: "on the left", "on the right", "in the center"
3. **액션 분리**: 각 캐릭터의 동작을 별도 문장으로
4. **이름 사용 금지**: "GAMER", "GPT" 대신 외형으로 설명

### 일관성 유지

- 동일 캐릭터는 **동일한 visual 설명** 사용
- 의상, 액세서리 변경 금지
- 배경이 바뀌어도 캐릭터 외형은 고정

## 10. 주의사항

1. **정리.md 수정 금지**: 기획안 원본 유지
2. **영어 프롬프트**: Veo 입력은 항상 영어
3. **한글 대사**: `audio_reference.dialogue_ko`에만 (영상 미포함)
4. **TODO.md 실시간 갱신**: 작업 시작/완료 시 상태 변경
5. **시리즈 일관성**: 캐릭터 visual 설명 3편 내내 동일하게 유지
6. **후편집 항목 구분**: 자막, 말풍선, 효과음, 선택지는 영상에 포함하지 않음
